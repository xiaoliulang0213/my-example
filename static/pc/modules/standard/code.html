java开发规范

代码规范
参考开发工具的代码格式化结果，开发工具推荐使用IDEA
日志规范

严禁使用System.out.print及e.printStackTrace记录日志

日志定义：public Log logger = LogFactory.getLog(getClass())，如果为静态类则public static Log logger = LogFactory.getLog(Xxx.class)

日志使用：日志-logger.log，调试信息logger.debug，警告信息logger.warn，错误信息logger.error，记录日志时做好日志级别的选择

rest开发规范

method规范
API的资源操作方法支持：GET、POST、PUT、DELETE等，我们使用GET和POST即可，GET表示查询，POST表示数据增删改
命名规则

Controller定义@RequestMapping("/api/moduleName")，接口方法定义@RequestMapping(value = "/xxx", method = RequestMethod.METHOD)

这样组合起来地址为：/api/moduleName/xxx

内部接口定义/api/xxx，公共接口定义/pub/xxx

前台通过util.get或util.post调用时，url为：/moduleName/xxx（/api无需定义，依赖于config.js中rest.baseUrl配置）

rest接口
private int status; private String code; private String message; private String debug; // 暂未使用 private String more; // 暂未使用 private String id; private String appId;

成功返回200，否则返回400、500等Http Code

失败返回数据格式为：{status: xxx, code: 'xxx', message: 'xxx', debug: 'xxx', appId: 'xxx}

status为http code

code为5位错误码，规则为：系统代码（P表示门户）+2位模块代码（自行划分）+2位错误代码

message为明确的可以直接反馈给用户的错误信息

debug为内部错误信息

appId为系统代码

html开发规范

vue显示

data-container属性对应的div增加v-cloak，在vue初始化完成之前隐藏改div，避免页面把{{}}显示出来，如果没有{{}}格式的数据，可以不用加这个class

显示数据，尽量用<div v-text="xxx"></div>的方式，而不是<div>{{xxx}}</div>

必填项
必填项前面必须有星号
label的使用
input输入框前应定义label for="id"，自定义标签通过属性attr定义id

js开发规范

引号的使用
js中字符串定义应使用单引号（'），如果遇到嵌套则外层单引号，内层双引号，如var x = '这是"测试"数据';
空值的判断
取任何属性之前应判断对象是否为空，比如var attr = var && var.attr;
控制台输出
开发过程中可以自行调试控制台输出，但不允许提交，更有效的调试方式是通过浏览器的开发者模式加断点
require.js

框架采用require.js来控制js依赖，常用依赖均已通过main.js引入（比如table、select、auto等），页面上可以不用再次添加依赖

但用到jquery、util、Const时需要通过require(['jquery', 'util', 'Const'], function($, util, Const) {}方式引入

main.js是主配置，如果js修改需要刷新缓存，需要设置require.config({urlArgs: '_=1.0.0'})，将1.0.0改一下

提示信息
统一提示信息调用为util.show()
